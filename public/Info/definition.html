Orienté objet {# Sécurité #} XSS (Cross-site scripting) : (coté client) C'est une faille de sécurité qui permet à un attaquant d'injecter dans un site web un code client malveillant. Ce code est exécuté par les victimes et permet aux attaquants de contourner les contrôles d'accès et d'usurper l'identité des utilisateurs. CSRF (Cross-site request forgery) : (coté serveur) Ce type d'attaque consiste simplement à faire exécuter à une victime une requête HTTP à son insu. Le but est de faire aller notre victime sur une page pour qu'il exécute les actions de la page, avec ses privilèges (généralement plus élevés que les nôtres). La victime doit être authentifié. S'en premunir à l'aide d un tokken CSRF qui sera hashé et stocké coté serveur. Injection SQL : Les attaques par injection de commandes SQL exploitent les failles de sécurité d’une application qui interagit avec des bases de données. L’attaque SQL consiste à modifier une requête SQL en cours par l’injection d’un morceau de requête non prévu, souvent par le biais d’un formulaire. Faille Upload : La faille upload est une faille permettant d'uploader des fichiers avec une extension non autorisée, cette faille est due à la mauvaise configuration du script d'upload ou à l'absence complète de sécurité. Mémoire tampon : La mémoire tampon (buffer en anglais) sert à stocker temporairement des données dans la mémoire vive ou dans le disque dur d’un ordinateur.
